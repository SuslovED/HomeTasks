# README.md

# Task1 - Вариант 1 (*) Калькулятор 1

## Описание варианта

Реализована клиент-серверная программа по варианту 1 ("Калькулятор 1") с использованием сокетов (AF_INET, SOCK_STREAM).

**Сервер:**
- Реализован по "Базовой модели": при подключении нового клиента создаётся дочерний процесс (`fork`), который его обслуживает. Родительский процесс продолжает принимать новые подключения.
- Дочерний процесс для каждого клиента хранит своё значение "инкремента" (число, на которое будут увеличиваться запросы).
- Поддерживаемые команды:
  - `+<число>` – установить новое значение инкремента для данного клиента. Ответ: `OK`
  - `<число>` – запрос на увеличение числа. Сервер возвращает `<число> + текущий_инкремент`
  - `\?` – запрос текущего значения инкремента
  - `\-` – корректное завершение работы клиента (сервер завершает дочерний процесс)
- Сервер игнорирует пустые сообщения и сообщения, состоящие только из пробелов.
- Поддерживаются различные окончания строк (`\n`, `\r`, `\r\n`).
- Обрабатываются сигналы:
  - `SIGCHLD` – для предотвращения появления процессов-зомби

**Клиент:**
- Подключается к серверу по заданному IP-адресу и порту (задаются в коде константами).
- Поддерживаемые локальные команды:
  - `\exit` – завершение работы клиента
- Все введенные пользователем строки отправляются на сервер с окончанием `\n`.
- Корректно обрабатывает сигнал `SIGINT` (Ctrl+C), закрывая соединение с сервером.

## Процесс сборки проекта

Сборка проекта производится с помощью утилиты `make`.

```bash
# Очистка предыдущей сборки (опционально)
make clean

# Сборка сервера и клиента
make all

# Или по отдельности
make server
make client
```

В результате сборки появятся исполняемые файлы `server` и `client`.

## Процесс запуска проекта

Запуск производится из командной строки.

### Запуск сервера

```bash
./server
```

Сервер запустится на порту 8080 (задано константой в коде).

### Запуск клиента

```bash
./client
```

Клиент подключится к серверу на localhost (127.0.0.1) порт 8080.

### Пример сессии

**Терминал 1 (сервер):**
```bash
$ ./server
Server on port 8080
```

**Терминал 2 (клиент):**
```bash
$ ./client
Connected to server
> 5
6
> +10
OK
> 5
15
> \?
10
> \-
Goodbye!
> \exit
```

## Содержание файлов проекта

- **`server.c`** – Исходный код сервера
  - `main()`: создание сокета, привязка, ожидание подключений, создание дочерних процессов
  - `handle_client()`: функция, выполняемая в дочернем процессе. Содержит логику обработки команд калькулятора
  - `handle_sigchld()`: обработчик сигнала SIGCHLD для уборки зомби-процессов

- **`client.c`** – Исходный код клиента
  - `main()`: подключение к серверу, основной цикл ввода-вывода
  - `handle_sigint()`: обработчик сигнала SIGINT для корректного завершения по Ctrl+C

- **`Makefile`** – Файл для автоматизации сборки проекта

- **`README.md`** – Данный файл с описанием проекта

## Используемые технологии

- **Язык:** C (стандарт C99)
- **Сборка:** GNU Make
- **Сетевые интерфейсы:** Сокеты Беркли (Berkeley sockets) домена `AF_INET`, тип `SOCK_STREAM` (TCP)
- **Системные вызовы:** `socket`, `bind`, `listen`, `accept`, `connect`, `fork`, `signal`, `waitpid`, `read`, `write`, `close`
- **Библиотеки:** Только стандартная библиотека C. Никаких сторонних библиотек не используется

## Текстовый протокол взаимодействия

### Формат сообщений

1. **Клиент → Сервер:**
   - Команды отправляются в виде текстовой строки, заканчивающейся символом `\n`
   - Пробельные символы в начале команды игнорируются сервером
   - Пустые строки игнорируются

2. **Сервер → Клиент:**
   - Ответы отправляются в виде текстовой строки, заканчивающейся символом `\n`

### Таблица команд

| Команда клиента | Ответ сервера | Описание |
|-----------------|---------------|----------|
| `+<число>` | `OK` | Установить инкремент |
| `+<нечисло>` | `Invalid` | Ошибка в команде + |
| `<число>` | `<результат>` | Увеличить число на инкремент |
| `\?` | `<текущий_инкремент>` | Запросить текущий инкремент |
| `\-` | `Goodbye!` | Отключиться от сервера |
| `любая другая строка` | `Unknown` | Неизвестная команда |

## Особенности реализации

- **Обработка сигналов:**
  - `SIGCHLD` – позволяет избежать накопления зомби-процессов при завершении клиентов
  - `SIGINT` – обеспечивает корректное завершение клиента по Ctrl+C

- **Обработка пробелов:**
  - Пробелы в начале команды игнорируются (например, `"  5"` обрабатывается как `"5"`)

- **Обработка пустых сообщений:**
  - Сообщения, состоящие только из пробелов, игнорируются

- **Проверка чисел:**
  - Используется `strtol` для точного определения, является ли строка числом (в отличие от `atoi`, который не отличает `"10"` от `"10abc"`)

## Известные ограничения

- Максимальная длина команды – 255 символов (ограничение буфера)
- Сервер не имеет команды для завершения (завершается по Ctrl+C)
- Порт и IP-адрес сервера задаются константами в коде, не через аргументы командной строки

## Завершение работы

- **Клиент:** `\exit` или `Ctrl+C`
- **Сервер:** `Ctrl+C` в терминале сервера
